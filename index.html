Flight Controller for Boeing 737-800

This project involved designing a controller and simulating a Boeing 737-800 aircraft using Simulink and FlightGear. The goal of this controller was to keep the airplane at a constant pitch angle. To develop the controller, I started by using principles of 3D dynamics to generate a state-space model for the aircraft. Next, using that state-space model, I used Simulink to generate a closed-loop block diagram with a PID controller to adjust the altitude in real time. The PID controller was tuned using Simulink's PID Tuner application. To verify the control system, a step input was fed into the closed loop and simulated with FlightGear.

nterplanetary Hohmann Transfer of Spacecrafts

Using the FreeFlyer software, I calculated and programmed a simulation to simulate the Hohmann transfer from Earth to Mars. The FreeFlyer software is a program designed for simulating the orbit of spacecrafts, with the programing language utilized being similar to Java. The process of modeling starts with calculating the Hohmann velocity, which is done using the axes of the orbits and gravitational parameter of the sun. Next, the phase angle is computed. With both of these and some additional setup in the FreeFlyer software, we can simulate our spacecraft's orbital transfer.

Rocket Design and Simulation

The main goal of this project was to practice both rocket design and computational fluid dynamics. Using OpenRocket, I designed a simple rocket, ensuring stability by selecting parts to make the center of gravity and center of pressure are towards the bottom of the rocket. Next, using ANSYS/OpenFOAM, I have set up a fluid dynamics simulation to attempt to calculate drag generated in my rocket design.

Optimal Control for Bipedal Robot

Using MATLAB and Simulink, I developed, tested, and simulated optimal control algorithms for a bipedal robot. The two control algorithms that I chose to implement were Model Predictive Control (MPC) and the Linear Quadratic Regulator (LQR) algorithm. With the Simscape Multibody MaTLAB add-on, I was able to develop graphs to display the results of each algorithm. From my results, the LQR algorithm provided better gait but worse path following, whereas the MPC algorithm stayed on its path better but had an awkward gait.

Reinforcement Learning for Quadcopter Control

This project applied reinforcement learning (RL) techniques to develop a control policy for a quadcopter. The Linear Quadratic Regulator (LQR) algorithm was used for the control of the quadcopter, and this control algorithm is tuned by modifying its Q and R matrices. The goal of this project is to find Q and R matrices that minimize mean position and velocity error for the quadcopter. While the RL model was programmed in Python, the simulation, control, and testing of the quadcopter is done in MATLAB.

Dynamic Path Planning of Autonomous Lawn Mower

I developed a programmable mower simulation using WeBots that utilized mapping and path planning algorithms to autonomously mow a lawn. For path planning, the Boustrophedon Cellular Decomposition (BCD) algorithm was used. Furthermore, LiDAR-based SLAM is being integrated into the simulator in order to allow for dynamic obstacle avoidance and rerouting.

Road Sign Detection Model

The main goal of this project was to develop a deep learning model that could identify road signs with high accuracy. In order to implement this, I chose to utilize Region-Based Convolutional Neural Networks (RCNN). The dataset used in the initial training had 877 images with annotations with images of four classes: speed limit signs, traffic lights, stop signs, and crosswalks. The first training of the developed algorithm achieved an accuracy of around 87%. At the moment, I am building on this project with the goals of improving accuracy, broadening the dataset to include more classes, and integrating OpenCV for real-time detection. ROS2 Humble is used to integrate all of the programs.

Real-Time Audio Synthesizer

I developed a program to implement audio effects in real time using Matlab and C. The effects implemented were an echo effect, the flanging effect, and a chorus effect. The Finite Impulse Response (FIR) and Infinite Impulse Response (IIR) to manipulate audio input and achieve desired effects. The algorithm was deployed onto an FPGA board to test and verify its correctness.

Five Stage MIPS Pipelined Processor

This project involved programming a five-stage pipelined processor in Verilog with the fetch, decode, execute, memory access, and writeback stages. In order to make this program more robust, stalling and hazard detection mechanisms were utilized to prevent errors with reading and writing data. To test and verify our program, we used assembly test scripts to run our program and handle standard and edge cases. Intel ModelSim was used to simulate the processor and debug our Verilog program.

Theremin Construction

Working with two others, I constructed a theremin device. A theremin is an instrument that is played by moving one's hands around it as opposed to touching it. The theremin construction involved soldering PCBs, wiring/assembling our boards, simulating our circuits with LTSpice, and testing/analyzing our results using an oscilloscope. The theremin contained the following components: motherboard, power supply, volume processor, pitch mixer, pitch processor.

Aerial Robotics Competition Teaml

Upon starting my graduate program at KSU, I decided to join ARCT. Our club's goal is to develop a lightweight autonomous aircraft for the SAE Aero Design challenge. I am a part of the GNC (Guidance, Navigation, Control) subteam, in which I primarily focus on our aircraft's communication and GPS systems. Our communication system relies on the use of two 900Mhz telemetry modules to strike a good balance of strength and range in our connection. For the GPS system, we are attempting to integrate multiple techniques in order to improve our accuracy as much as possible, with the main optimization being the usage of real-time kinematics (RTK). In addition to RTK, we also are attempting to implement post-processing kinematics (PPK) and the Extended Kalman Filter (EKF) in conjunction with an Inertial Measurement Unit (IMU) in order to perform sensor fusion.
